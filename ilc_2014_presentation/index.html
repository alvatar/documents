Getting Started

with SchemeSpheres

Origin of the project

the potential of...

  1. providing self-contained set of libraries and standard SRFIs
  2. integrating mobile and desktop platforms development
  3. while exposing Gambit's facilities and flexibility

...additionally,

aim at providing

  1. project structure templates and build tasks for all platforms, without hiding the details and allowing full customization per-project.
  2. project construction automation, in all development platforms, for the different targets available in each.
  3. dependency handling and some form of package specification, so the system favours reusability and code sharing.
  4. a common documentation system, manuals and guides.

Requirements

  1. Complete control over the workflow and build process, as opposed to typical module systems.
  2. Take advantage of multiple backends and execution models.
  3. Support for syntax-rules.
  4. Support for low-level macros (define-macros/syntax-case/rsc-macro-transformer)
  5. Module system or namespaces

cond-expand

should be declared manually


(##spheres-include core: base-macros)

;; Witout any added expansion feature
(cond-expand
 (my-condition
  (display "my-condition is met\n"))
 (else
  (display "no condition is met\n")))

;; After adding "my-condition" expansion feature
(add-cond-expand-feature! my-condition)

(cond-expand
 (my-condition
  (display "my-condition is met\n"))
 (else
  (display "no condition is met\n")))

include evals the code, used to make macros available at compile-time and the REPL

The
Spheres

Core Sphere

  • Assert
  • Base macros (SRFIs and general use macros)
  • FFI generation and utilities
  • Functional programming
  • Lazy (SRFI-45)
  • Match
  • Lightweight testing (SRFI-78)

Energy Sphere

Extension to Core's infrastructure

  • Easy argument processing / Purely functional
  • Basic contracts
  • System error codes
  • Exceptions
  • I/O extensions
  • Let-multiple (SRFI-71)
  • Localization (SRFI-29)

Energy Sphere

(continuation)

  • Logging utilities
  • Profiling utilities
  • Remote debugging / REPL server
  • External representation of data with shared structures (SRFI-38)
  • Micro templating system
  • API for test suites (SRFI-64)
  • Time procedures (SRFI-19)

Fabric Sphere

Algorithms

  • Compare procedures (SRFI-67)
  • Eager comprehensions (SRFI-42)
  • List procedures (SRFI-1) + Additional list procedures
  • Reservoir sampling, random selection
  • Shuffle algorithms
  • Sort and Merge algorithms (SRFI-95)
  • Algorithms for Streams (SRFI-41)
  • Algorithms for u8 Vectors
  • Algorithms for Vectors (SRFI-43)

Fabric Sphere

Data Structures

  • Arrays (SRFI-63)
  • Multi-dimensional Arrays (SRFI-25)
  • SRFI-69 compatible table (although Gambit provides it's own tables implementation)
  • Streams (SRFI-41)

Fusion Sphere

Multiplatform development

  • Mobile support and Sfusion generators
  • Platform-specific modules
  • Remote Spheres module management

Strings Sphere

String handling and related

  • String library (SRFI-13)
  • Character sets (SRFI-14)
  • String formatting (SRFI-28, SRFI-48)
  • u8 Vector strings

Codec Sphere

Codecs

  • base64
  • codec utils

Crypto Sphere
